# PropertyVisualizer ルール

## 概要
間取り図から3D可視化・リフォーム費用計算を行うツール開発プロジェクト。
ブラッシュアップしながら実用化を目指す。

## 構想
- 間取り図（画像）から部屋構成・面積を解析
- リフォーム費用の概算計算（マスター価格ファイル連携）
- Before / After イメージ生成
- 3D表示・360°ビュー（将来）
- 仮想ツアー動画（将来）

## 開発方針
- まずClaude Code上でロジック・挙動を固める
- 動作確認が取れたらStreamlitアプリ化して発行・共有
- ブラッシュアップしながら機能を拡張していく

## フェーズ構成
- Phase 1：リフォーム費用シミュレーター（Python）← 現在
- Phase 2：間取り図解析（Claude Vision API）
- Phase 3：Before/After画像生成（外部AI連携）
- Phase 4：3D・360°・動画（Three.js / Blender等）
- Phase X：Streamlitアプリ化・デプロイ（各フェーズ完了後）

## Phase 1 設計メモ

### 目的
「業者見積りが明らかにおかしくないか」をサクッと判断する概算ツール。
精密な積算ではなく、**相場とのズレを素早く検知すること**が目的。

### 入力
| 項目 | 備考 |
|------|------|
| 専有面積（㎡） | 必須 |
| 間取り（1K〜4LDK等） | ハウスクリーニング費・部屋数の算出に使用 |
| パターン選択 | A / B / C / Custom |

### 面積算出ロジック（一律ルール）
- 天井高：2.4m（固定）
- 床面積 = 専有面積
- 天井面積 = 専有面積
- 壁面積 = 専有面積 × 2.5（業界慣例の概算係数）

### パターン定義
| パターン | 内容 | 想定用途 |
|---------|------|---------|
| A | 必須（原状回復）：クロス・CF・クリーニング | 退去後の最低限リフォーム |
| B | A ＋ 競争力向上：LED・温水洗浄便座・モニターホン・鍵交換 | 賃貸募集力アップ |
| C | B ＋ 商品化：キッチン・UB・給湯器・洗面台 | 売却・フルリフォーム |
| Custom | 全項目をY/Nで個別選択 | → 将来：Streamlitでトグル/スライダーに置換 |

### マスター価格ファイル
- ファイル：`20260612_master_prices.csv`
- 単価は施工費込み・既存撤去含む目安価格（2026年時点）
- 調整はCSVを直接編集するだけでよい

### 学習・精度向上の仕組み
- 計算実行のたびに `renovation_log.csv` へ自動保存
- 実績（業者最終請求額）を後からログに追記
- 将来：パターン別乖離率レポートスクリプトで単価調整を提案

ログ構造：
```
date, area, madori, pattern, estimated, actual, ratio, note
```

乖離率（actual/estimated）をパターン別に集計し、単価を段階的に補正していく。
Bの乖離 − Aの乖離 = B追加項目（LED等）の誤差、という形で絞り込みも可能。

### 判定基準（業者見積りとの比較）
| 概算比 | 判定 |
|--------|------|
| 130%以上 | ⚠ 要確認（概算より30%以上高い） |
| 110〜130% | △ やや高め |
| 70〜110% | ○ 概ね適正 |
| 70%以下 | ✓ 割安 |
